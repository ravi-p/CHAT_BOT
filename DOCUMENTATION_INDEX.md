## üìö Feature Documentation

### Multiple Chat Sessions
- **What:** Independent chat sessions with isolated context
- **Why:** Support multiple concurrent conversations
- **How:** See [ARCHITECTURE.md - Sessions](ARCHITECTURE.md#-services-layer)
- **Example:** [QUICKSTART.md - How to Use](QUICKSTART.md#how-to-use)

### Scalable Routing
- **What:** Modular API design for easy extension
- **Why:** Add features without changing existing code
- **How:** See [ADDING_FEATURES.md](ADDING_FEATURES.md)
- **Example:** Three examples provided in feature guide

### Session Isolation
- **What:** Each session has independent documents/memory
- **Why:** No data leakage between sessions
- **How:** Diagram in [DIAGRAMS.md](DIAGRAMS.md#data-flow-multi-session-isolation)

### Persistent Storage
- **What:** Sessions survive application restarts
- **Why:** Data durability
- **How:** See [ARCHITECTURE.md - Storage](ARCHITECTURE.md)

### Professional UI
- **What:** Clean, modern interface with session management
- **Why:** Better user experience
- **How:** Component docs in [PROJECT_STRUCTURE.md](PROJECT_STRUCTURE.md)

## üîó Cross References

### Terms & Definitions
- **Service**: Business logic layer, independent and testable
- **Router**: API endpoint layer, delegates to services
- **Session**: Independent chat context with unique ID
- **Vectorstore**: FAISS index for document embeddings
- **RAG**: Retrieval Augmented Generation (retrieve docs + generate)
